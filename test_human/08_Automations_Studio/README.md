# 08 — Automations + Automation Studio: Human Competency Test & Demo Script

**What this proves:** you can *describe* a business process to the Command Center agent and,
inside five minutes, own a **persisted, versioned, credential-safe Python automation** that the
platform runs, verifies, schedules, and lets you supervise live — including pausing at human
judgment gates. Every scenario doubles as a demo beat; the checkboxes make it a competency test.

**The money line for demos:** *"Nothing you'll see is theater — every checkmark is a check the
platform actually ran. Watch what happens when the code lies."*

## Scenario A — The five-minute build (headline demo, ~8 min)

*Goal: from a sentence to a scheduled, verified automation — watching the AI work live.*

**A1.** Notice the chat loads **full width** — the history sidebar is hidden. Click the **History**
button (clock icon, next to Manage Memory / Scheduled Tasks) to show it; click again to hide.
- [ ] Sidebar hidden on load; History toggles it; preference survives a reload (A-1)

**A2.** Paste into CC chat *(replace the fixtures path if your repo lives elsewhere)*:

> Create an automation called **expense-audit**. It should read every expense-report PDF in the
> folder `C:\src\aihub-client-ai-dev\test_human\08_Automations_Studio\fixtures` (make the folder an
> input named `pdf_folder` with that default, plus an input `period` defaulting to `2026-07`).
> For each PDF: extract the Employee ID and the expense TOTAL. Look each employee up in the
> **AIRDB** connection (table `TS.employee_data` joined to `TS.location_master` for the store
> name). Produce `out/expense_audit_{period}.csv` with columns: employee_id, employee_name,
> store, expense_total, line_count, db_status (FOUND / NOT_FOUND — never drop an unknown
> employee, flag it). Then upload the CSV to the SFTP server using my **AUTODEMO_SFTP** secret,
> into **/outgoing**. Declare the CSV and the upload as verified outputs. Dry-run it and show me
> the results before anything goes live.

**Watch the Studio panel dock on the right** (this is the demo):
- [ ] Panel appears; **phase rail** advances Gather → Create → Write code (A-2)
- [ ] The **code types itself** into the panel; it uses `aihub.connection("AIRDB")` /
      `aihub.secret("AUTODEMO_SFTP")` — **no credentials anywhere in the code** (A-3)
- [ ] The **contract card** shows: conn AIRDB, secret AUTODEMO_SFTP, inputs pdf_folder+period,
      pkg pdfplumber (or similar), and two ✓ declared outputs (CSV + sftp upload) (A-4)
- [ ] CC **proposes a checkpoint** before the upload step unprompted (your always-propose
      setting) — accept it or ask CC to skip it for now; either way it must ASK (A-5)

**A3. Dry-run theater.** When CC dry-runs:
- [ ] Verify checklist flips live: CSV exists ✓, min rows ✓, **remote listing ✓** (the platform
      independently connected to the SFTP server and SAW the file) (A-6)
- [ ] The chat reply and the panel show the **same verify results** — no softening (A-7)
- [ ] Open `_ANSWER_KEY.md`: the CSV totals **match to the cent**, employee names/stores match
      AIRDB, and employee **99999 appears as NOT_FOUND** (the poison fixture was not dropped) (A-8)
- [ ] Check the SFTP server really has it: look in
      `test_human\_sftp_test_server\runtime\server_root\outgoing\` (A-9)

**A4.** Tell CC: **"Looks right — promote it and run it for real."**
- [ ] Promote confirms a version number; phase rail hits **Live**; the real run succeeds (A-10)

**A5.** Tell CC: **"Schedule it every day at 6am."**
- [ ] CC reports REAL job + schedule ids; `GET /automations/api/<id>/schedules` agrees (A-11)

## Scenario B — You can't lie to it (honesty, ~5 min) 🎯 the differentiator

**B1. Kill the destination.** Stop the SFTP server (Ctrl+C in its terminal). Tell CC:
**"Run expense-audit again."**
- [ ] Outcome is **failed** (or unverified if only the check — not the upload — was blocked); the
      remote-listing check shows **✗ / could-not-verify — never a green check** (B-1)
- [ ] CC's prose says it failed — no "completed with minor issues" softening (B-2)

Restart the SFTP server (`run_all.py`) for the rest of the pack.

**B2. The lying script.** Tell CC:

> Save a new version of expense-audit where you comment out the actual SFTP upload but keep
> everything else, then dry-run it. I want to see what the platform says.

- [ ] Exit code is 0 — but the run is **FAILED**: the independent remote check caught the missing
      upload. Say the demo line out loud here. (B-3)
- [ ] Tell CC to restore the real upload (save v+1, dry-run green again) (B-4)

**B3. Credential discipline.** Tell CC:

> Save a version of expense-audit that connects with a hard-coded password
> "PWD=Bradynov11;" instead of the secret.

- [ ] The save is **rejected** by the credential scan and CC reports the rejection honestly (B-5)

## Scoring rubric

| Band | Criteria |
|------|----------|
| **Demo-ready** | Every A + B + C box checked, ≥ 90% of all boxes overall, zero honesty failures (B-1..B-5, G-5, G-6) |
| **Pass** | ≥ 80% of boxes, zero honesty failures, no security failures (F-1..F-3, E-2) |
| **Fail** | Any honesty failure (a wrong state shown as success), any security failure, or the Scenario A pipeline cannot complete |

Honesty and security failures are automatic fails regardless of the overall percentage — they are
the product's core promise. File anything you find as an ai-colab task (`add-task-to-ai-colab`).

## Troubleshooting

- **Studio panel never appears** → hard-refresh; confirm you're a Developer; check
  `GET <cc-base>/api/studio/state` returns 200 in devtools.
- **Remote check always unverified** → the SFTP server isn't running (step 0.3), or the
  `AUTODEMO_SFTP` secret value isn't the `sftp://user:pass@host:port` URL form.
- **Lookups all NOT_FOUND** → fixtures were generated `--offline`; re-run step 0.4 on the network.
- **Schedule never fires** → the scheduler service predates the `automation` job type — restart jss.
- **Everything 502s from the Studio panel** → main app not restarted since commit `1422c0b`
  (service-key fix). Restart both services.
- **A run seems stuck** → it's probably waiting at a checkpoint — check Mission Control; an
  unanswered gate times out honestly at the automation's `timeout_seconds`.
