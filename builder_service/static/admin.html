First-Class Agent Execution

Agents are treated as equal execution options alongside direct API actions.

1 User sends a "build" request
→
2 Planner sees APIs + Agents as options
→
3 Plan includes agent steps: {domain: "agent"}
→
4 Execute calls agents during plan execution

How the Planner Chooses

  • • Simple CRUD operations → Direct API actions (agents.create, workflows.create)
  • • Complex design tasks → Delegate to specialized agents (workflow_agent, report_agent)
  • • Mixed tasks → Plan can include both API calls and agent delegations

Agent specializations help the planner understand when to delegate. Agents can have multi-turn conversations to gather requirements.
