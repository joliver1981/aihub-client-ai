Extraction Configuration
Workflow contract — what the workflow receives and must return
Input variables (passed automatically by the compliance engine when the workflow starts):
  • document_path (string) — absolute path to the uploaded PDF on disk
  • document_set_id (string) — id of the set this document belongs to
Output variable — your workflow must set extraction_result before completing. If it is missing or empty, the Requirements view will be empty for this document version. extraction_result must be one of:
Shape A — schema-style (same shape produced by schema-based extraction):
{
  "fields": {
    "category.subcategory.field_name": {
      "value": "the extracted value",
      "sources": [{ "pages": [page_number] }]
    },
    ...
  }
}
Shape B — explicit row list (lets you populate Value, Severity, Source Page, and Confidence directly):
[
  {
    "category":         "service_levels",
    "subcategory":      "availability",
    "requirement_text": "Monthly uptime of not less than 99.95% ...",
    "specific_value":   "99.95%",
    "severity":         "high",
    "source_page":      1,
    "confidence":       0.95
  },
  ...
]
Timeout — workflow execution is awaited synchronously and times out after DOC_PROCESSING_TIMEOUT_MINUTES (default 120 minutes). Plan accordingly for long-running pipelines.
